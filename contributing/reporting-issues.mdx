Bug reports and feature requests help make kChat better for everyone. Both are welcomed via GitHub Issues at [github.com/korynthian/chatroom/issues](https://github.com/korynthian/chatroom/issues). Clear, detailed reports make it much easier for the maintainer to understand, reproduce, and address the problem — so the guidelines below are worth a few minutes of your time before you submit.

<Tip>
  Search [existing issues](https://github.com/korynthian/chatroom/issues) before opening a new one. Someone may have already reported the same bug or requested the same feature, and adding a comment to an existing issue is more helpful than creating a duplicate.
</Tip>

## Reporting a bug

<Steps>
  <Step title="Open the Bug Report template">
    Go to the [new issue page](https://github.com/korynthian/chatroom/issues/new) and select **Bug report** to load the template.
  </Step>
  <Step title="Describe the bug clearly">
    Provide a concise description of what the bug is. Include what you **expected** to happen and what **actually** happened instead. The clearer this contrast, the faster the issue can be diagnosed.
  </Step>
  <Step title="List steps to reproduce the issue">
    Write out the exact sequence of actions that triggers the bug, for example:

    1. Go to the room list page
    2. Click on a room name
    3. Type a message and press Enter
    4. Observe the error in the console

    Reproducible steps are the single most valuable piece of information in a bug report.
  </Step>
  <Step title="Include your browser and OS details">
    State your **browser name and version** (for example, Chrome 124, Firefox 126, Safari 17) and your **operating system** (for example, Windows 11, macOS 14, iOS 17). If you are reporting from a smartphone, also include the device model. Version numbers matter — behaviour can differ between releases.
  </Step>
  <Step title="Attach screenshots if relevant">
    If the bug produces a visible error, an unexpected layout, or a console message, a screenshot or screen recording makes it much easier to understand at a glance. Attach any images directly to the GitHub issue form.
  </Step>
  <Step title="Submit the issue">
    Review your report, then click **Submit new issue**. The maintainer will triage it and follow up in the thread.
  </Step>
</Steps>

## Requesting a feature

<Steps>
  <Step title="Open the Feature Request template">
    Go to the [new issue page](https://github.com/korynthian/chatroom/issues/new) and select **Feature request** to load the template.
  </Step>
  <Step title="Describe the problem your feature would solve">
    Explain what you are currently unable to do, or what friction you encounter. For example: "I'm always frustrated when I have to scroll all the way up to find older messages because there is no search." Framing the request around a real problem helps evaluate whether and how to solve it.
  </Step>
  <Step title="Describe the solution you'd like to see">
    Give a clear description of the behaviour or UI you have in mind. Be as specific as you can — sketches, mockups, or references to similar features in other apps are all helpful.
  </Step>
  <Step title="Mention any alternatives you've considered">
    If you thought of other ways to solve the same problem and decided against them, say so and explain why. This shows that you've thought the request through and gives the maintainer useful context when evaluating options.
  </Step>
  <Step title="Submit the request">
    Add any additional context or screenshots that might help, then click **Submit new issue**. Feature requests are not guaranteed to be implemented, but all suggestions are read and considered.
  </Step>
</Steps>
